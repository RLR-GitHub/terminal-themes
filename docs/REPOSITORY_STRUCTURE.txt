FILE DESCRIPTIONS
=================

Core Files:
-----------
README.md              - Main repository documentation with setup instructions
LICENSE                - MIT License text
QUICKSTART.md          - Fast setup guide (under 60 seconds)
CONTRIBUTING.md        - Guidelines for contributors
CHANGELOG.md           - Version history and release notes
.gitignore            - Git ignore patterns
install.sh            - Automated installation wizard

Theme Scripts:
--------------
All theme scripts in themes/ follow the same structure:
  - init_term()     : Initialize terminal settings
  - cleanup()       : Restore terminal on exit
  - rain()          : Main animation function
  - alert()         : Flash alert messages
  - SYMBOLS         : Character set for rain
  - ALERTS          : Alert message array
  - Main loop       : Handles --init or infinite mode

Demo Files:
-----------
rory-terminal-themes.html : Interactive web showcase
  - Live canvas animations
  - One-click script copying
  - Responsive design
  - All 5 themes displayed

Documentation:
--------------
docs/CUSTOMIZATION.md     : Advanced theme customization
docs/TROUBLESHOOTING.md   : Common issues and fixes
docs/THEMES.md            : Detailed theme descriptions
docs/API.md               : Script function reference
docs/FAQ.md               : Frequently asked questions


QUICK SETUP FOR DEVELOPERS
============================

1. Clone the repository:
   git clone https://github.com/yourusername/terminal-themes.git
   cd terminal-themes

2. Make scripts executable:
   chmod +x install.sh
   chmod +x themes/*.sh
   chmod +x scripts/*.sh

3. Test a theme:
   ./themes/matrix-hacker.sh --init

4. Run tests:
   ./tests/test-all.sh

5. Start local demo server:
   cd demo
   python3 -m http.server 8000
   # Visit: http://localhost:8000/rory-terminal-themes.html


CREATING A NEW THEME
=====================

1. Copy template:
   cp themes/matrix-classic.sh themes/matrix-newtheme.sh

2. Edit theme:
   - Update SYMBOLS with your characters
   - Change color ranges in rain() function
   - Customize ALERTS array
   - Test thoroughly

3. Add documentation:
   - Update README.md theme table
   - Add entry to CHANGELOG.md
   - Create screenshot in assets/

4. Submit PR:
   - Create feature branch
   - Commit changes
   - Open pull request


INSTALLATION LOCATIONS
======================

After installation, files will be placed at:

~/matrix.sh                 - Selected theme script
~/.bashrc or ~/.zshrc       - Shell configuration (appended)
~/.bashrc.backup.TIMESTAMP  - Backup of original config

RELEASE PROCESS
===============

1. Update CHANGELOG.md with new version
2. Tag release: git tag -a v3.0.0 -m "Release v3.0.0"
3. Push tag: git push origin v3.0.0
4. Create GitHub release with:
   - Release notes from CHANGELOG
   - Attached files: install.sh, all themes
   - Screenshots and demos
